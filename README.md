## вҡЎ Energy Hygiene вҖ” A BEATEK Standard

The BEA Aura Console runs 24/7. It is your private cloud, your inference node, your security monitor, and your passive income engine вҖ” all at once. That means power quality directly affects:

- **GPU-Fi reliability** вҖ” ML trainers renting your compute expect uptime; an outage is a conduct violation under GPU-Fi Network Standards


### Green Energy Return вҖ” Why This Helps the Grid

A UPS does more than protect your hardware. When paired with clean energy sources, it participates in a responsible energy loop:

- **Battery buffer** absorbs peak draw spikes вҖ” your GPU-Fi burst load never hits the grid raw
- **Brownout absorption** вҖ” instead of pulling unstable current and wasting it as heat, the UPS regulates draw
- **Clean shutdown on outage** вҖ” no phantom loads, no dirty reconnects, no wasted grid cycles
- **For solar and self-powered setups** вҖ” the UPS acts as the bridge between your generation and your load, smoothing the handoff and returning cleaner draw patterns to any shared grid connection

This is recycled, regulated energy returned to the grid in a cleaner state than it arrived. That is green energy hygiene.

### BEAScanner Contract

Every pillar implements a BEAScanner subclass:
1. Receives domain-specific events
2. Maps them to E[0вҖ“31] BEA states
3. Emits transitions via BEA_Pulse
4. Tracks ScanResult: `bea_state`, timestamp, event detail

### Integration Pattern

Each pillar's `integration.py` provides:
- Per-pillar **Adapter** classes (e.g., `HealthVaultAdapter`, `GridLedgerAdapter`)
- **Broadcaster** class that emits to BEA_Pulse
- Optional specialized adapters (e.g., `LedgerRefundAdapter`, `PillarContextBridge`)

### Async Architecture

- All pillars use Python `asyncio` internally
- BEA_Pulse broker is async-first with optional Redis backend
- Test framework: `anyio` вҖ” use `@pytest.mark.anyio` per async test function
- No `pytest-asyncio`, no class-level markers, no async fixtures
